# 来财协作面板 · 设计方案 v0.1

> 两人异步协作面板。第一版只做三件事：**看回测报告 / 记策略想法 / 记交易计划**。
> 不碰回测触发、不碰实盘。先解决沟通损耗，出第一个可用版。

---

## 0. 项目定位

| 项 | 值 |
|---|---|
| 目标用户 | 你 + 朋友（2人，异步协作） |
| 部署形态 | NAS + H5（PWA），公网 HTTPS 访问 |
| 项目目录 | `D:\workstations\TelecomQt` |
| 与 LaiCai 仓库关系 | 本项目是 LaiCai 回测引擎之上的「协作前端 + 轻后端」。回测仍用 LaiCai 的 `self_strategy/` 纯 pandas 脚本，本项目消费其产出的报告。 |
| 核心诉求 | 把「口头 + 截图 + 聊天记录」式的异步协作，换成「同一个网页面板读写」式异步协作 |

---

## 1. 整体架构

```text
┌──────────────────────────────────────────────────────┐
│  手机 / 电脑浏览器（PWA H5，可加到主屏幕）                │
│   报告详情 / 想法表单 / 计划看板                          │
└────────────────────────┬─────────────────────────────┘
                         │ HTTPS（公网）
┌────────────────────────┴─────────────────────────────┐
│  NAS: Cloudflare Tunnel + 鉴权                         │
│        ↓                                              │
│  Nginx/Caddy 反代                                     │
│        ↓                                              │
│  FastAPI 后端（Docker, restart:always）                │
│   ├── /reports  读实验报告                             │
│   ├── /ideas    策略想法 CRUD                          │
│   ├── /plans    交易计划 CRUD                          │
│   └── /assets   静态图表                               │
│        ↓                                              │
│  SQLite（想法/计划/实验元数据）+ 文件系统（报告/图表）    │
└────────────────────────┬─────────────────────────────┘
                         │ 消费
┌────────────────────────┴─────────────────────────────┐
│  LaiCai 回测产出（你在电脑跑 self_strategy → 报告+JSON+PNG）│
└──────────────────────────────────────────────────────┘
```

**关键边界**：除最底层「跑回测」外，面板和 LaiCai 解耦。LaiCai 输出报告文件，面板只读文件。

## 2. 技术选型

### 后端
- **Python 3.12 + FastAPI**：异步、自带 API 文档、轻量，和 LaiCai 同语言
- **Uvicorn**：ASGI server
- **SQLite**：两人数据量极小，无需 MySQL；存想法/计划/实验元数据
- **Pydantic**：数据校验（与 LaiCai PythonGO 风格一致）

### 前端（v0.1 求快，零构建）
- **Alpine.js + Pico.css**：无构建步骤，NAS 上直接跑静态文件；交互够用
- **ECharts**：收益曲线、K线、买卖点标注（H5 渲染最强）
- **PWA**：`manifest.json` + service worker，可「加到主屏幕」像 App
- 预留升级路径：交互变复杂时可迁 Vue 3 + Vite

### 部署 / 网络
- **Docker** 跑后端（`restart: always` 自动重启，化解你担心的停机风险）
- **公网访问**：**Cloudflare Tunnel**（免费、不用公网IP、不用开端口、自动 HTTPS）← 首选
  - 备选：frp + 自有域名 + Let's Encrypt（需公网IP/DDNS）
- **鉴权**：Cloudflare Access（零信任，免费，最省心）；或后端简单 session 登录（两人共用密码）

## 4. 数据模型

### 4.1 策略想法 Idea
| 字段 | 说明 |
|---|---|
| name | 策略名 |
| instrument | 品种（rb / ag / ...） |
| timeframe | 周期（H2 / H4 / D1） |
| entry_condition | 开仓条件（文字，可选结构化勾选） |
| exit_condition | 平仓条件 |
| stop_loss / take_profit | 止损 / 止盈 |
| volume | 手数 |
| note | 备注 |
| status | 新想法 / 已转回测 / 已采纳 / 已搁置 |
| author / created_at | 谁、何时 |

### 4.2 交易计划 Plan
| 字段 | 说明 |
|---|---|
| instrument / direction | 品种 / 方向（多/空） |
| planned_entry | 计划入场价或区间 |
| stop_loss / target | 止损 / 目标 |
| planned_date | 计划执行时间 |
| status | 计划中 / 已执行 / 已放弃 |
| actual_entry / actual_pnl | 执行后回填 |
| note / author / created_at | |

### 4.3 回测实验 Experiment（元数据，指向 LaiCai 报告）
| 字段 | 说明 |
|---|---|
| strategy_name / instrument / timeframe | |
| report_path | LaiCai 报告目录路径 |
| spec_snapshot | 参数快照（YAML/JSON） |
| metrics_json | 收益/胜率/回撤/夏普（解析自报告） |
| note | 你和朋友的批注 |

> Experiment 不存交易明细本体——交易明细从 LaiCai 报告目录现读，保持单一真相源。

## 5. 报告解析策略（关键）

LaiCai 现在输出 Markdown。要让面板结构化展示（指标卡 + 交易明细表），有两条路：

- **方案A（推荐）**：让 LaiCai 回测脚本**额外输出一份 `experiment.json`**（指标 + 交易明细数组），面板直接读。改动极小（在 `_generate_report.py` 加个 `json.dump`）。
- 方案B：后端解析现有 Markdown。脆，不推荐。

→ **选 A**。新老报告都结构化，面板不碰 Markdown 解析。

## 6. 页面设计

### 页面 1：报告列表 + 详情
- 列表：每条实验（策略名 / 品种 / 日期 / 收益 / 胜率），按时间倒序，按品种筛选
- 详情：
  - 指标卡：总收益、胜率、盈亏比、最大回撤、夏普、交易笔数
  - 收益曲线图（ECharts）
  - **逐笔交易明细表**：时间 / 品种 / 方向 / 开平 / 价格 / 手数 / 盈亏 / 触发原因
  - 批注区（你和朋友都能加）

### 页面 2：策略想法
- 按状态分组列表 + 新增想法表单（字段见 4.1）
- 提交后 status=新想法；你看过后标「已转回测」或「已采纳」

### 页面 3：交易计划
- 看板式（计划中 / 已执行 / 已放弃 三列）
- 新增计划表单（字段见 4.2）
- 执行后可回填实际入场和盈亏

## 7. 与 LaiCai 的衔接

v0.1 不碰回测触发，流程：
1. 你在电脑跑 LaiCai 的 `self_strategy` 回测 → 产出 Markdown + `experiment.json` + PNG
2. 产物同步到 NAS 的 `backend/data/reports/`（rsync / 网盘 / 手动拷）
3. 后端 `report_loader` 扫描该目录，自动登记为新 Experiment
4. 朋友在面板看到新报告

**未来 v0.2「一键回测」**：面板按钮 → 后端调 LaiCai 脚本 → 自动产出。可行，因 LaiCai 回测是纯 pandas，NAS 上能直接跑。

## 8. 部署步骤（NAS）

1. NAS 装 Docker（多数 NAS 系统自带）
2. backend 打 Docker 镜像，`docker-compose up -d`（含自动重启）
3. frontend 静态文件由 Nginx/Caddy 托管（或直接 FastAPI 托管）
4. 公网：装 `cloudflared`，建 Tunnel，绑你的子域名，自动 HTTPS
5. 鉴权：Cloudflare Access（零信任，免费）或后端登录页
